================  tablet.sh — cheatsheet  ================
cd /home/sebastian/ADB-tablet

--- INTERACTIVE CONSOLE (optional) ---
  ./tablet.sh ui              full-screen console (devices + actions + hotkeys)
                              builds itself on first run; needs the Rust toolchain
  tab                         same console from anywhere (symlink in ~/.local/bin)

--- CONNECT (one tablet = ONE single transport) ---
  ./tablet.sh usb    [name]   permanent cable    -> USB runtime   (most stable)
  ./tablet.sh qr     [name]   QR pairing         -> WiFi runtime
  ./tablet.sh wifi   [name]   known endpoint     -> WiFi runtime
  ./tablet.sh use             numbered menu: pick a tablet / change its type
  ./tablet.sh                 reconnects ALL according to their saved type
  ./tablet.sh status          see each tablet and how it is connected

--- OPERATE ---
  ./tablet.sh cap    [name]   screenshot -> /tmp/<tablet>-<date>.png
  ./tablet.sh url             pick URL(s) (e.g. 1 3 4) + tablet(s) (e.g. 1 2 or 'all')
                              then pick the browser (it lists the ones installed on the device)

--- MANAGE ---
  ./tablet.sh rm              pick device(s) and remove them from the registry
  ./tablet.sh proxy [cmd]     proxy control: start | stop | status | logs
  ./tablet.sh off [name]      disconnects one tablet, or EVERYTHING: tunnels, WiFi adb,
                              proxy, adb server (enrollment survives -> ./tablet.sh)

--- RESPONSIVE / DEBUG ---
  ./tablet.sh size            emulate other screens: phone, phone-small, tablet-8,
                              tablet-10, fold-open, custom (WxH dpi), reset
  ./tablet.sh inspect [name]  Chrome DevTools bridge -> chrome://inspect on your PC

--- DEBUG / CHROME ---
  ./tablet.sh clear           clears Chrome cache+cookies+SERVICE WORKER (one/several/'all')
  ./tablet.sh stop            closes Chrome (one/several/'all')
  ./tablet.sh logs   [name]   live Chromium logcat (Ctrl+C to exit)
  ./tablet.sh rec [name] [s]  records the screen -> /tmp (Ctrl+C, or stops after [s] seconds)

-------------------------------------------------------
FIRST TIME
  deps: ./setup.sh          (one shot: adb, node, nmap, ..., Rust, console, 'tab')
  USB : plug in the cable  ->  ./tablet.sh usb
  QR  : tablet -> Wireless debugging -> "Pair with QR"  ->  ./tablet.sh qr

ON THE TABLET (Chrome): open the SAME url as on your local machine
  http://localhost:5173/#/login
  http://localhost:5050

UNPAIR / START FROM SCRATCH
  delete the line in devices.env  (or empty the file)  +  adb disconnect

-------------------------------------------------------
NOTES
  - USB = cable plugged in, the tunnel travels over the cable (does not drop with WiFi).
  - WiFi = do not leave the cable plugged in or the tunnel sticks to USB (UsbFfs).
  - 'clear' is your friend: if the tablet shows something old/cached, it is
    almost always the service worker -> ./tablet.sh clear and reload.
  - closing the console does NOT drop connections -> ./tablet.sh off does
    (registry kept; ./tablet.sh rm is the one that unenrolls).
  - Port 80 (Moodle) can NOT be reverse-forwarded by adb -> it goes through the Vite proxy.
  - devices.env -> source of truth: name | serial | ip:port | type(usb|wifi)
